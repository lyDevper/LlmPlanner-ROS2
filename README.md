# LlmPlanner-ROS2

**LlmPlanner-ROS2** is a ROS 2 (Humble) workspace that integrates Large Language Models (LLMs) to act as a high-level cognitive planner for mobile robots. 

By leveraging LangChain and ReAct agent architectures, the core node translates natural language prompts—ranging from precise engineering commands to vague directional instructions—into structured, actionable trapezoidal trajectory parameters.

## 📦 Packages

This workspace is divided into two primary packages:

* **`llm_pack`**: Contains the core logic, launch files, and the primary LangChain agent node (`llm_node.py`).
* **`llm_pack_interface`**: Contains the custom ROS 2 interface definitions required by the planner, specifically the `/traj_context` message and the `/LlmPrompt` service.

## 🛠️ Prerequisites & Dependencies

### System Requirements
* **OS:** Ubuntu 22.04
* **ROS 2:** Humble Hawksbill
* **Build Tool:** `colcon`

### Python Dependencies
The `llm_node.py` relies on the LangChain ecosystem to orchestrate the LLM and tool-calling functions. Install the required Python packages:

```bash
pip install langchain langgraph langchain-google-genai langchain-openai
```

### Environment Variables
The node is currently configured to use Google's Gemini 2.5 Flash model for fast, tool-calling compatible inference. You must expose your API key to the terminal environment before running the node:

```bash
export GOOGLE_API_KEY="your_google_api_key_here"
```
*(Note: If the code is reverted to use OpenAI, ensure `OPENAI_API_KEY` is exported instead).*

The tutorial link on ubuntu env var: https://www.xda-developers.com/set-environment-variable-in-ubuntu/

I (Lucky) personnally used the "sudo nano ~/.bashrc" method

## 🏗️ Building the Workspace

1. Source your ROS 2 installation:
   ```bash
   source /opt/ros/humble/setup.bash
   ```
2. Navigate to the root of your workspace and build the packages:
   ```bash
   colcon build --packages-select llm_pack_interface llm_pack
   ```
3. Source the local workspace overlay:
   ```bash
   source install/setup.bash
   ```

## 🚀 Usage

### 1. Starting the Planner Node

You can start the LangChain node using either `ros2 run` or the provided launch file.

**Option A: Using the Launch File**
```bash
ros2 launch llm_pack llm_pack.launch.py
```

**Option B: Running the Node Directly**
```bash
ros2 run llm_pack llm_node.py
```

### 2. Monitoring the Trajectory Output

Open a new terminal, source the workspace, and echo the context topic to observe the trajectory parameters generated by the LLM:

```bash
ros2 topic echo /traj_context
```

### 3. Sending Prompts to the Robot

You can interact with the robot by calling the `/LlmPrompt` service. The LLM is capable of interpreting both vague, human-centric instructions and precise mathematical commands.

**Example 1: Vague/Relative Instructions**
```bash
ros2 service call /LlmPrompt llm_pack_interface/srv/String "{prompt: 'Move yourself to 14 Oclock slowly, about the distance of 3 man steps.'}"
```

**Example 2: Precise/Absolute Instructions**
```bash
ros2 service call /LlmPrompt llm_pack_interface/srv/String "{prompt: 'Move the robot to x=5.0, y=2.0, z=0.0. Use a cruising velocity of 1.5 m/s and an acceleration of 0.035 m/s^2.'}"
```

## 🧠 Node Details: `llm_node.py`

The `llm_node` acts as a service server for incoming prompts and a publisher for outgoing trajectory data.

* **Services:**
    * `/LlmPrompt` (`llm_pack_interface/srv/String`): Accepts a `prompt` string from the user and returns a `response` string from the LLM. You can send the prompt `"reset history"` to clear the agent's conversational memory.
* **Publishers:**
    * `/traj_context` (`llm_pack_interface/msg/TrajContext`): Publishes the parsed spatial goal (x, y, z), constant phase velocity (v_const), and acceleration (a) to downstream trajectory generation nodes.
* **Agent Logic:** Uses `create_react_agent` with a custom `publish_context` tool. The LLM infers the physical constraints of the robot and formats its response to trigger the ROS 2 publisher automatically.

## 📨 Custom Interfaces (`llm_pack_interface`)

The planner relies on custom message and service definitions to communicate between the LangChain agent and the lower-level robot controllers. 

### 1. Trajectory Context Message (`msg/TrajContext.msg`)
This message carries the high-level trapezoidal trajectory parameters generated by the LLM. It dictates *how* the robot should move to its destination.

```ros2
# TrajContext.msg

# The target spatial goal for the robot
geometry_msgs/Vector3 s_goal

# The constant phase cruising velocity (m/s)
float32 v_const

# The acceleration/deceleration rate (m/s^2)
float32 a

# The initial state/pose of the robot (defaulted to 0,0,0 by the LLM)
geometry_msgs/Vector3 q_init

# Trajectory phase constant 
int8 part
int8 CONSTANT=1 # Example constant used in the node
```

### 2. LLM Prompt Service (`srv/String.srv`)
A simple string-in, string-out service used to send natural language instructions to the robot and receive the agent's textual feedback or success confirmation.

```ros2
# String.srv

# Request: The natural language instruction from the user
string prompt
---
# Response: The confirmation or error message from the LangChain agent
string response
```
